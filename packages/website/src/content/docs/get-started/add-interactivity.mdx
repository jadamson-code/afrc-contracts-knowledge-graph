In [Part 3](/get-started/style-the-graph/), you styled the graph declaratively and added a hover effect. In this final step, you will highlight a node's neighbors on hover and add a search bar that focuses the camera on results.

## The goal

When the user hovers a node, its neighbors stay colored while everything else fades to grey. A search bar lets the user type a node name, highlights matches, and animates the camera to the selected node.

This requires **custom state** — sigma's built-in `isHovered` flag only applies to the single hovered node. To grey out everything else, you need a way to mark which nodes and edges are "active" (part of the hovered neighborhood).

## Define custom state types

Add custom state interfaces that extend sigma's built-in state. Place these above your `main()` function:

```typescript
import type { BaseEdgeState, BaseGraphState, BaseNodeState } from "sigma/types";

interface NodeState extends BaseNodeState {
  isActive: boolean;
}

interface EdgeState extends BaseEdgeState {
  isActive: boolean;
}

interface GraphState extends BaseGraphState {
  hasActiveSubgraph: boolean;
}
```

Then pass them as generic parameters to Sigma:

```typescript
const renderer = new Sigma<object, object, object, NodeState, EdgeState, GraphState>(
  graph,
  document.getElementById("container")!,
  {
    /* ... styles ... */
  },
);
```

## Add conditional styles for the active subgraph

Update the styles to grey out nodes and edges that are not active when a subgraph is highlighted:

```typescript
styles: {
  nodes: [
    {
      color: {
        attribute: "cluster",
        dict: clusterColors,
        defaultValue: "#999",
      },
      size: { attribute: "score", min: 50, max: 250, minValue: scoreExtents.min, maxValue: scoreExtents.max },
      label: { attribute: "label" },
    },
    {
      when: "isHovered",
      then: { labelVisibility: "visible" },
    },
    {
      when: (_attrs, state, graphState) =>
        graphState.hasActiveSubgraph && !state.isActive && !state.isHovered,
      then: { color: "#f6f6f6", label: "" },
    },
    {
      when: "isActive",
      then: { labelVisibility: "visible" },
    },
  ],
  edges: [
    { color: "#ccc", size: 5 },
    {
      when: (_attrs, state, graphState) =>
        graphState.hasActiveSubgraph && !state.isActive,
      then: { color: "#f6f6f6" },
    },
  ],
},
```

The third node rule uses a function predicate: it checks the `graphState.hasActiveSubgraph` flag and the node's own `isActive` state. When a subgraph is active and the node is not part of it, it fades to grey and hides its label.

## Wire up hover highlighting

After creating the renderer, add event listeners that set the active subgraph on hover:

```typescript
function setActiveSubgraph(activeNodes: Set<string> | null) {
  const active: string[] = [];
  const inactive: string[] = [];
  graph.forEachNode((node) => {
    (activeNodes && activeNodes.has(node) ? active : inactive).push(node);
  });
  renderer.setNodesState(active, { isActive: true });
  renderer.setNodesState(inactive, { isActive: false });

  const activeEdges: string[] = [];
  const inactiveEdges: string[] = [];
  graph.forEachEdge((edge) => {
    const [source, target] = graph.extremities(edge);
    (activeNodes && activeNodes.has(source) && activeNodes.has(target) ? activeEdges : inactiveEdges).push(edge);
  });
  renderer.setEdgesState(activeEdges, { isActive: true });
  renderer.setEdgesState(inactiveEdges, { isActive: false });

  renderer.setGraphState({ hasActiveSubgraph: !!activeNodes && activeNodes.size > 0 });
  renderer.refresh({ skipIndexation: true });
}

renderer.on("enterNode", ({ node }) => {
  const neighbors = new Set(graph.neighbors(node));
  neighbors.add(node);
  setActiveSubgraph(neighbors);
});

renderer.on("leaveNode", () => {
  setActiveSubgraph(null);
});
```

`setNodesState` and `setEdgesState` batch-update state flags efficiently. The `refresh({ skipIndexation: true })` call tells sigma to re-evaluate styles without re-computing the spatial index (since no positions changed).

Try hovering a node — its neighbors should stay colored while the rest fades out.

## Add a search bar

Add a search input to your HTML, above the container:

```html
<div id="search">
  <input type="search" id="search-input" list="suggestions" placeholder="Search for a node..." />
  <datalist id="suggestions"></datalist>
</div>
<div id="container"></div>
```

Style it however you like (absolute positioning works well). Then wire it up in `index.ts`, after creating the renderer:

```typescript
const searchInput = document.getElementById("search-input") as HTMLInputElement;
const suggestions = document.getElementById("suggestions") as HTMLDataListElement;

// Populate the autocomplete list
suggestions.innerHTML = graph
  .nodes()
  .map((node) => `<option value="${graph.getNodeAttribute(node, "label")}"></option>`)
  .join("\n");

searchInput.addEventListener("input", () => {
  const query = searchInput.value.toLowerCase();

  if (!query) {
    setActiveSubgraph(null);
    return;
  }

  // Find nodes whose label matches the query
  const matches = graph
    .nodes()
    .filter((n) => (graph.getNodeAttribute(n, "label") as string).toLowerCase().includes(query));

  if (matches.length === 1) {
    // Exact match: highlight the node and animate the camera to it
    const match = matches[0];
    const neighbors = new Set(graph.neighbors(match));
    neighbors.add(match);
    setActiveSubgraph(neighbors);

    const position = renderer.getNodeDisplayData(match);
    if (position) {
      renderer.getCamera().animate(position, { duration: 500 });
    }
  } else if (matches.length > 0) {
    // Partial matches: highlight all of them
    setActiveSubgraph(new Set(matches));
  } else {
    setActiveSubgraph(null);
  }
});
```

Type a name like "Gephi" or "Graph" in the search bar. Matching nodes highlight, and when there is a single match, the camera animates to it.
