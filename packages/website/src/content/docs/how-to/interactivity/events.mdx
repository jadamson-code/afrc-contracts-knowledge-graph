Sigma.js emits events when users interact with the graph. You can listen for clicks, hovers, and other mouse events on nodes, edges, and the stage (the background area).

## How it works

### Available events

**Node events:** `enterNode`, `leaveNode`, `downNode`, `clickNode`, `rightClickNode`, `doubleClickNode`, `wheelNode`

**Edge events:** `enterEdge`, `leaveEdge`, `downEdge`, `clickEdge`, `rightClickEdge`, `doubleClickEdge`, `wheelEdge`

**Stage events:** `downStage`, `clickStage`, `rightClickStage`, `doubleClickStage`, `wheelStage`

### Enabling edge events

Edge events are disabled by default for performance reasons. Enable them with the `enableEdgeEvents` setting:

```typescript
const renderer = new Sigma(graph, container, {
  enableEdgeEvents: true,
});
```

### Listening for events

```typescript
renderer.on("clickNode", ({ node, event }) => {
  console.log("Clicked node", node);
  console.log("Mouse position", event.x, event.y);
});

renderer.on("enterNode", ({ node }) => {
  // User is hovering this node
});

renderer.on("clickStage", ({ event }) => {
  // User clicked the background
});
```

Every event payload includes an `event` object with `x`, `y` coordinates and the `originalEvent` (the native `MouseEvent` or `TouchEvent`).

For the full events reference, see the [Events reference](/reference/events/).
