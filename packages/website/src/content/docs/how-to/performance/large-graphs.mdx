Sigma is designed to render graphs with thousands of nodes and edges at interactive frame rates using WebGL. Here are some tips for working with large graphs.

## Built-in optimizations

Sigma applies several optimizations automatically:

- **Label culling**: Only labels that fit without overlap are rendered. Control the threshold with `labelRenderedSizeThreshold`.
- **Hide on move**: During camera movements, edges and labels can be hidden for smoother interaction:
  ```js
  settings: {
    hideEdgesOnMove: true,
    hideLabelsOnMove: true,
  }
  ```

## Layout strategies

For large graphs, pre-computing the layout is faster than running it live:

```js
import forceAtlas2 from "graphology-layout-forceatlas2";

// Run layout synchronously (blocking but fast)
const settings = forceAtlas2.inferSettings(graph);
forceAtlas2.assign(graph, { settings, iterations: 100 });
```

For interactive layouts, use the web worker version:

```js
import FA2Layout from "graphology-layout-forceatlas2/worker";

const layout = new FA2Layout(graph, { settings: forceAtlas2.inferSettings(graph) });
layout.start();
// layout.stop() when done
```

## Reducing visual complexity

For very large graphs, consider reducing what's displayed:

- Use `nodeReducer` / `edgeReducer` to filter or simplify elements
- Adjust `labelRenderedSizeThreshold` to show fewer labels
- Use simpler node shapes (circles are fastest)
- Avoid expensive layers (images, piecharts) on large graphs
